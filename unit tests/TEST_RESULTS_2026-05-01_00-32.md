## Notable additions since the 2026-04-29 23:43 full-suite run

This was a busy cycle — 22 commits, three feature areas:

### Energy blade deflect path (5 commits)

* **Zone-routing fix** (`6756b84`) -- routes Zone 2 +
  Star Maze projectile-vs-player handlers through
  `_try_melee_deflect`.  Diagnosed via the temporary JSONL
  telemetry (now removed in `4be44ac`): a 126.8 s session with
  216 swings produced 0 deflect attempts because the deflect
  hook was only wired to Zone 1.  Six tests pin the new routes.
* **Lifetime bump** (`4e0f822`) -- `MELEE_SWING_LIFETIME`
  0.25 s → 0.35 s closes the ~33 % "blade idle between swings"
  gap observed in the deflect log without breaking
  one-hit-per-swing semantics.
* Telemetry harness added then removed once the diagnosis was
  complete.

### Energy Pickaxe (10 commits)

The fourth weapon -- a melee-style mining tool that damages
asteroids only.

* `319f781` introduces it: same swing cadence as the lightsabre,
  PICKAXE_DAMAGE = 20 (+15 for Debra), in its own
  `_active_pickaxe` slot so the deflect path keeps keying off
  `_active_blade` only.  12 tests pin cycle position, blade
  lifecycle, AOE damage, Debra bonus, no enemy damage,
  no-deflect-while-pickaxe-active.
* Sprite tuning across `0936aa8` → `073cc32` settles the asset
  on `assets/ai generated/energypickaxe2.png` (1024×1024,
  PICKAXE_SCALE 0.10, PICKAXE_TEX_ANGLE_OFFSET +45°,
  PICKAXE_HANDLE_OFFSET_PX (323, -353)).
* `073cc32` adds `head_offset_px` parameter on `MeleeBlade` and
  routes the AOE hit zone through `head_pos` (rotated head
  position) instead of the sprite centre, so the effective
  range tracks the business end as the swing arcs.

### Bot stack (7 commits)

* **50/50 mining-weapon dice** (`820af76`) -- on each fresh
  S_MINE entry, autopilot rolls between Mining Beam and Energy
  Pickaxe.  Sticky for the entire mining session.  6 tests.
* **Hold-distance for pickaxe** (`dba6a9b`) -- new
  `_do_hold_distance` helper maintains
  `PICKAXE_HOLD_DISTANCE_PX = 100` from the asteroid centre
  with a 20 px dead-band so the bot doesn't ram while mining
  with a melee weapon.  4 tests.
* **Starter-base trigger** (`01ce0d6`) -- new S_BUILD state +
  `bot_builder.py` module.  When iron ≥ 1000 AND no asteroids
  within 400 px AND no aliens within 600 px, autopilot POSTs
  `/build_starter_base` once.  In-process handler places
  Home Station + Service + Power Receiver + Solar Array 2 +
  Repair Module + 2× Turret 2 (max free-place 300 px).  11
  tests pin the gate, the one-shot semantics, and the build
  sequence ordering.
* **Combined launcher** (`99c34ba`) -- `bot_run.py` calls
  `bot_kickoff.main()` then `bot_autopilot.main()` in one
  terminal.  Both child scripts remain standalone entry points.
* **Zone-aware /state** (`b28ee1b`) -- routes alien + asteroid
  extraction through `draw_logic._minimap_enemies` /
  `_minimap_obstacles` so the bot sees the same lists the
  minimap reads (Zone 2 / Star Maze / warp aliens were
  invisible to the bot before).  3 tests.
* **Edge-stuck recovery** (`4159025` + `d9ef18a`) -- 1.5 s
  rolling-window stuck detection with two-condition exit
  (min duration AND ship clear of 500 px from every edge);
  spiral re-anchors to world centre on stuck so SEARCH doesn't
  re-pin; log is throttled to once per 5 s.  9 tests.

The ~53 net new tests (1371 → 1424) all pass in the fast suite
and add no measurable cost to `combat_assist.tick`,
`bot_api.get_state`, or `_do_auto` per the performance budget
tests.  Soak runtime within 6 s of the prior baseline.
