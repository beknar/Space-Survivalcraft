## Basic Ship rebuild soak — leak investigation close-out

This soak is the one that failed at +95.6 MB on 2026-06-20 (over the
75 MB budget), triggering the #274 texture-atlas leak fix, the #276
audio-cleanup fix, and a deep memory investigation.

| Cycle | Mem growth | Result | Notes |
|---|---:|:---:|---|
| 2026-06-16 | +50.0 MB | PASSED | |
| 2026-06-18 | +62.0 MB | PASSED | |
| 2026-06-20 00:09 | **+95.6 MB** | **FAILED** | texture-atlas leak |
| 2026-06-20 08:48 | +69.0 MB | PASSED | post #274 texture fix |
| **2026-06-22 09:46** | **+0.0 MB** | **PASSED** | post #276 audio fix |

**Investigation conclusion (carry-forward closed):** the residual
growth after the texture fix was **not** a discrete code leak.  It was
a mix of (a) an accelerated-time pyglet audio-player backlog — the
wall-clock-aged cleanup couldn't reap SFX players when the soak runs
hundreds of frames per wall-clock second; fixed in #276 by also ageing
by sim-time — and (b) pymalloc arena retention from per-frame
allocation churn (survives a full `gc.collect()`, scales ~1.2 KB/frame,
reads higher in cold-allocator isolated runs than in the warm
full-suite context).  This cycle confirms the rebuild path is clean:
0 MB growth in the real full-suite context with both fixes in place.
