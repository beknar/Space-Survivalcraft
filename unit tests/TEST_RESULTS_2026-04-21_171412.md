### ⚠️ Systemic regression in soak suite

All but two of 58 soak tests failed. FPS across the board dropped from the normal 100-200 range to **10-25 FPS**. Frame counts over the 5-min runs dropped from ~25 000 – ~45 000 (prior runs) to **1 500 – 5 000**. Perf tests (which run 60 frames in a fresh GameView) still showed normal FPS (200-500), so the regression is either (a) in a code path the soak churn tick hits but perf tests don't, or (b) inter-test state leakage between the soak fixtures.

The only soak test to start fast was **TestSoakZone1**, which opened at 322 FPS and decayed monotonically to 23.2 FPS over 5 min (322 → 138 → 83 → 54 → 30 → 20 → 22 → 25 → 30 → 19 → 23). Every subsequent soak test started at ≤27 FPS — the degraded state persists across the suite, suggesting a global leak.

Likely culprit: last two commits on main (`1a42aa1` "Nebula boss rams through asteroids and drops normal loot" and `1eda700` "Tests covering this session's boss + drop tweaks") touched `sprites/nebula_boss.py`, `update_logic.py`, and `zones/zone2_world.py`. These ran on every tick of `run_soak` but not on the 60-frame perf tests.

- **Two Nebula boss integration failures** (`test_gas_cloud_damages_player_and_slows`, `test_cone_damages_player_while_inside`) point at the same feature area.
