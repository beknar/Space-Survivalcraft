### `test_soak_bot_modules.py::TestStuckDetectHistoryBounded::test_history_stays_bounded_over_30k_ticks`

PR #58 (Bot: add long-window net-progress gate to stuck-detect)
extended `record_position`'s history retention from
`STUCK_DETECT_WINDOW_S = 1.5` s to
`STUCK_DETECT_LONG_HISTORY_S = 5` s — the long history feeds the
new long-window net-progress gate in `detect_stuck`.  At 10 Hz
that's ~50 samples in steady state, but the soak test still
asserted the old `≤ 20` upper bound.

Fix: bumped the assertion to `≤ 60` (5 s × 10 Hz + boundary
buffer for samples stamped exactly at the cutoff).  Updated the
class docstring to point at the new retention constant and
explain the new bound.  No code change — this is a stale
test assertion that wasn't updated when PR #58 landed.
