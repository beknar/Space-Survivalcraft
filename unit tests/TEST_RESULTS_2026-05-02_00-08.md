## Notable additions since the 2026-05-01 00:32 full-suite run

This was a 17-commit cycle, mostly bot stack maturation plus the
three refactor commits the prior report's "candidates" section
flagged.

### Refactors (3 commits)

* **BotState dataclass** (`8cd84de`) — five module-level globals
  (_fsm, _spiral_state, _stuck_state, _mining_weapon_pick,
  _build_done) consolidated into a single dataclass with a
  `reset()` method.  Adding a new state field is now one line +
  one reset entry, not five `global` declarations across files.
* **Enemy-projectile-hit helper** (`a0221ca`) — five duplicated
  "deflect → damage → shake → bump" call sites collapsed into
  one `apply_enemy_projectile_hit(gv, proj, *, bump_volume=...)`.
  Future enemy projectile sources get deflect for free.
* **Parameterised blade lifecycle** (`2c7daa5`) — six near-byte-
  identical functions (`_ensure_melee_blade` /
  `_remove_melee_blade` / `update_melee_blade` plus the parallel
  pickaxe trio) replaced with three generic functions
  parameterised on a tiny `_BladeKind` config.

### Bot behaviour additions (8 commits)

* **BUILD_SEEK + spiral-no-fire** (`8066d40`) — bot now actively
  walks toward less-cluttered space when iron is met but the area
  isn't clear (instead of waiting for a clear pocket to drift in
  by chance), and the spiral search no longer fires the mining
  weapon at empty space.  Single 800 px clear-area gate spans
  asteroids + aliens + pickups + buildings.
* **Centre-lockout** (`4f9d0fc`) — after a stuck-escape, the FSM
  is locked into SEARCH for 30 s so MINE / GATHER / BUILD can't
  immediately retarget the same edge object.  Breaks the
  observed "stuck → escape → re-target same edge object → stuck"
  infinite cycle.
* **Main-thread queue** (`d4b9d93`) — `submit_to_main_thread()` +
  `pump_main_thread_queue()` lets HTTP handlers safely run
  GL-backed mutation (sprite spawn, building placement) on the
  GL-context thread.  Diagnosed by a play-test where every
  starter-base attempt returned `GL_INVALID_OPERATION (0x1282)`.
* **Multi-phase build** (`26c3659`) — single POST now does Phase
  1 (starter base) + Phase 2 (deposit ship resources to station)
  + Phase 3 (west extension + Basic Crafter), 11 buildings + a
  resource transfer in one main-thread call.
* **Duplicate-build / stuck-in-base / max-count fixes**
  (`28d2e70`) — `_act_build` early-returns on `build_done`
  (prevents 6 redundant POSTs during MIN_DWELL); STARTER_BASE
  shifted 200 px north of the player so the ship doesn't end up
  trapped inside the Home Station; defensive max-count guard in
  `bot_builder` (place_building doesn't enforce max — that's
  build-menu UI only).
* **Deposit ALL resources + S_DEPOSIT state** (`14c48b9`) —
  deposit walks every item_type in ship inv (iron, copper,
  blueprints, anything) and transfers each individually with
  partial-add safety.  New S_DEPOSIT FSM state triggers when
  ship has ≥ 200 iron OR any blueprint, navigates to the home
  station, POSTs the deposit, 30 s cooldown.

### Other (4 commits)
