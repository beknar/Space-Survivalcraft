**The Boss-phase + Asteroid Zone 1 + Build-menu drops are
flagged for next-cycle attention** but every test still
passes its FPS cap.  None of the refactors in #171–#186
touched Boss combat or wheel-scroll code paths directly,
so the most likely cause is host-machine load variance
during the 5.5-hour run rather than a real code regression.

The Boss-rotation context in particular is sensitive to the
order in which the three phase contexts execute (each
inherits the prior's GL state), so a slightly different
pytest collection order between cycles can shift these
numbers by ±30 % without any code change.  Worth re-running
the Boss subset in isolation next cycle for a cleaner read.
