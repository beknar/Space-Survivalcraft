## Behaviour fixes shipped this cycle (2026-05-03 → 2026-05-04)

In rough chronological order, all driven by analysis of
`bot_io/autopilot_telemetry.jsonl` JSONL captures from real
game sessions:

* **bot_autopilot 3-module split** —
  `bot_autopilot_telemetry.py` (139 lines), `_navigation.py` (321
  lines), `_blacklist.py` (159 lines).  `bot_autopilot.py` itself
  shrank 2443 → 1983 lines (-19%).  Original API preserved via
  re-export shims; ~70 test references continue to work without
  change.

* **update_logic 2-module split** — `update_logic_sfx.py` (160
  lines), `update_logic_zone_effects.py` (227 lines).
  `update_logic.py` itself shrank 2102 → 1765 lines (-16%).  All
  ~25 `from update_logic import` call sites in collisions /
  draw_logic / combat_helpers / input_handlers continue working.

* **SEARCH-stuck false-fire fix (3 sub-fixes)** —
  - `_do_auto` skips stuck-detect when `_fsm["state"] == S_SEARCH`.
  - Spiral `stop_radius` reduced 120 → 40 px.
  - `_do_goto(brake_on_arrival=False)` for spiral search.
  Net: ~30 false-fire stuck events per session → 0.

* **Consumable-phase deadlock fix** —
  - New `CONSUMABLE_PHASE_IRON_THRESHOLD = 500` (down from sharing
    the module phase's 2000).
  - `DEPOSIT_IRON_THRESHOLD` lowered 200 → 100.
  - Auto-flip `consumable_phase_started = True` when install
    queue empties + iron ≥ 500 (latches the phase against
    transient iron dips).

* **HUNT mode** — new `S_HUNT` state.  Triggers when no asteroid
  is visible AND an alien is within `HUNT_RANGE_PX` (3000 px).
  Lower priority than ENGAGE (still fires defensively on close
  threats), higher priority than IDLE_AT_BASE / SEARCH.  Action
  reuses `_act_engage`.

* **IDLE_AT_BASE mode** — new `S_IDLE_AT_BASE` state.  Triggers
  when no asteroid + no alien in HUNT_RANGE_PX + Home Station
  exists.  Bot navigates to within `IDLE_AT_BASE_RADIUS_PX`
  (600 px after the cluster-oscillation fix) of HS and idles.
  Falls back to SEARCH when no HS exists yet (early game).

* **IDLE_AT_BASE cluster-oscillation fix** — initial 300 px
  radius put the bot inside dense station building clusters
  (HS + 10 placed buildings).  Aggregated potential field shoved
  the bot around, fired stuck-detect 12× in 5 minutes.  Fixes:
  - radius 300 → 600 (stop_radius now 480, well past any building's
    80 px field range);
  - S_IDLE_AT_BASE joins S_SEARCH on the stuck-detect exempt list.
