* **Auto-cascade refactor → formal FSM** (`6fc81b5`):
  - `bot_autopilot.py` — five-state FSM (ENGAGE / GATHER /
    REGEN / MINE / SEARCH) with asymmetric enter/exit

    (`unit tests/test_bot_autopilot_fsm.py`) pin hysteresis
    bands, dwell behaviour, and ENGAGE preemption.
  - Existing autopilot logic tests updated to match new
    40 % / 60 % shield bands.

* **Melee-commit dice on fresh engagement** (`a91a6e8`):
  - `bot_combat_assist.py` — per-engagement
    `MELEE_COMMIT_CHANCE = 0.5` dice roll fires on the tick
    that transitions no-threat → threat; on hit, the
    Energy Blade is locked for the rest of the engagement
    (with `MELEE_LOCK_HOLDOVER_S = 0.6 s` grace for brief
    target loss).  The lock is exposed as
    `state.assist.melee_engaged` so the autopilot can
    close to `MELEE_STOP_RADIUS_PX = 50 px` instead of the
    ranged 380 px stand-off.
  - 12 new dice + lock + grace tests in
    `unit tests/test_bot_combat_assist.py`.
  - 3 movement tests in
    `unit tests/test_bot_autopilot_fsm.py` cover the
    autopilot's stop-radius switch when committed.

The ~30 net new tests (1335 → 1365) all pass in the fast
suite and add no measurable cost to `_do_auto` /
`combat_assist.tick` per the performance budget tests.
