## Behaviour fixes shipped this cycle (2026-05-02 → 2026-05-03)

In rough chronological order, all driven by analysis of
`bot_io/autopilot_telemetry.jsonl` JSONL captures from real
game sessions:

* **NE/SW corner turret placement** — Turret 2s in the bot
  starter base now sit on the NE / SW corners of the Home
  Station at the maximum 300 px free-place radius, widening
  effective coverage vs. the prior straight east/west placement.

* **Post-build craft + install workflow** — three-phase serial
  queue runs after the starter base: 6 module crafts (armor
  plate → engine booster → shield booster → shield enhancer →
  damage absorber → broadside, gated by 2000 station iron +
  every blueprint deposited), 4 installs (broadside →
  shield_booster → shield_enhancer → armor plate), then 5×
  repair pack + 5× shield recharge consumable crafts.  Bot
  gathers between visits while the 60 s craft timer ticks down.

* **Boundary potential field** — per-axis repulsion vector
  pointing away from world edges is blended into every
  `_do_goto` heading, deflecting smoothly so the bot routes
  around corners rather than pinning.  Falls back to pure
  repulsion when the goto exactly cancels the field.

* **Building potential field** — same mechanic for player-built
  station structures; corners stack two contributions
  automatically.  Range (80 px) sits well inside the
  deposit/install/craft action ranges (200/250/200 px) so the
  bot still completes every station-side action.

* **Repulsion-vector escape burst** — the stuck-escape now
  heads along the combined boundary + building repulsion
  vector instead of toward world centre.  Falls back to world
  centre only when neither field is active.

* **Pickup blacklist** — when stuck-detect fires while in
  S_GATHER, the pickup is added to a 60 s TTL blacklist so
  the FSM stops re-targeting unreachable pickups.

* **Asteroid blacklist** — same mechanic for S_MINE (60 s TTL,
  40 px radius).

* **Build-done unconditional short-circuit** — moved to top of
  `_choose_next_state` so it fires every tick regardless of
  which higher-priority state wins the dispatch (was buried
  below GATHER, never executed when bot was actively gathering).

* **Centre-lockout removal** — the 30 s SEARCH-only post-escape
  suppression window was redundant once the potential field
  deflected proactively.  Kept the position-history watchdog +
  escape burst + spiral re-anchor as the reactive backstop.

* **Iron-only deposit gate** — removed the OR-blueprint
  shortcut that produced wasteful return trips with single
  blueprints; blueprints now ride along on the iron-triggered
  run.

* **Asteroid chase distance cap (2000 px)** — MINE-vs-SEARCH
  gate filters asteroids beyond 2000 px so the bot doesn't
  commit to long obstacle-laden cross-map trips.  Sticky
  chase commitment (`_state.chase_committed`) keeps the cap
  dropped once a far-asteroid chase is committed, preventing
  the FSM from bouncing SEARCH ↔ MINE every MIN_DWELL_S.

* **SEARCH-giveup escape hatch (60 s)** — drops the chase cap
  after 60 s of continuous SEARCH so the bot eventually
  commits to whatever's nearest.

* **Slowed SEARCH spiral angle advance (4°/tick from 8°)** —
  the prior tangential target speed (~280 px/s at 200 px
  radius) was faster than the ship could rotate to face,
  causing the bot to perpetually re-orient without thrusting.

* **Escape-exit spiral re-anchor at current position** — the
  prior re-anchor fired on stuck-detect with world-centre as
  the anchor, but the home station was usually built near
  world centre, so the next SEARCH cycle targeted right back
  through the building cluster.  Now anchors at the bot's
  clear-space landing position post-escape.

* **Rotation-aware stuck-detect** — stuck-detect now requires
  both small position change AND small heading change.  The
  SEARCH spiral's "reach target → brake → rotate to next
  target" cycle was firing the watchdog every 1.5 s during
  normal operation because position-only detection couldn't
  distinguish rotation from being pinned.

* **Test telemetry isolation** — autouse fixture in
  `unit tests/conftest.py` monkey-patches
  `bot_autopilot._telemetry_init` and `_telemetry_log` to
  no-ops during the test run, stopping the test suite from
  appending ~120 events per pytest invocation to the live
  `bot_io/autopilot_telemetry.jsonl` operator analysis file.
