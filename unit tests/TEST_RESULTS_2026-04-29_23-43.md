2026-04-29 00:29 report -- the doorway-midpoint test depends
on `next(iter(maze.doorways))` dict iteration order plus stale
sprite back-references, so it flips deterministically only
when launched fresh in a particular order.  Production drone
code is correct; this is purely a fixture / pyglet state
artefact.
