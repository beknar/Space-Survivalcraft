### Soak observations

- **Two real crashes (NebulaBossShip.radius)** — `sprites/nebula_boss.py:211` references `self.radius`, which the class doesn't define. Both nebula-boss soak tests die on the very first tick. The nebula-boss *performance* tests (entries 71–73 above) pass because those exercise a different code path (the cone-fire branch never triggers at their positioning). Simple fix: set `radius = BOSS_RADIUS` (or the appropriate value) on `NebulaBossShip`.
- **One real perf floor breach** — Build-menu wheel-scroll sits at ~51 FPS with repeated 31.1 FPS dips, matching the corresponding perf test (51.3 FPS vs 40 FPS floor). The scroll + wheel path is the suite's slowest non-crashing path right now.
- **One single-sample dip** — Dialogue character rotation hit 39.0 FPS at the 3.0 min sample; everything else in that test was 133–140 FPS and RSS was flat. Classic stutter.
- **Warm-up grace window is doing its job** — last run's three START-sample failures (T menu Zone 2 minimap, telemetry Zone 2, ...) would have been failures with the old rule but now cleanly pass because the grace window excludes START.
- **Several tests end near the floor (40–43 FPS)**: BossPhase3, BossPhaseRotation, BossInvulnerablePlayer, StationShield+AIFleet, Ellie-exhaustive, Telemetry Zone 2 — consistent with the prior run's thin-margin list. All still pass.
- **Warp soaks remain memory-dominant** — Meteor +720 MB, Lightning +1122 MB, Gas +678 MB across 2 min each. The Lightning growth crept up vs the prior run (+834 → +1122 MB) but still under its per-test override.
- **Telemetry record_frame overhead = 4.63 µs/call** (well under the 25 µs target; last run was 5.17 µs).
- **Integration regression:** `TestBossSpawn::test_boss_spawns_when_all_conditions_met` fails — `check_boss_spawn` no longer produces a boss for a harness that sets every listed precondition. Worth investigating whether a new precondition was added without updating the test harness, or whether the spawn logic regressed.
