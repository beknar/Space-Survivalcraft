## Triage

### Unit suite — 1 pre-existing flake

`test_soak_base_gc.py::TestGetRssMb::test_repeated_calls_return_close_values`
fails in the mixed unit run and **passes in isolation** (0.38 s
real time).  This is the same RSS-stabilization flake called out
in PR #75's TEST_RESULTS_2026-05-09 and every subsequent PR
description -- the assertion is `abs(rss_a - rss_b) < 5 MB` and
the natural RSS variance under load swings past that envelope.
Not a regression; documented as known flake.

### Soak suite — 4 memory-threshold failures

All four failed soak tests are **memory-threshold** failures, not
FPS failures (FPS levels are healthy across the board).  Three
remain failed in isolation; one passes in isolation but fails in
the mixed run.

Three real (small) memory regressions and one ordering flake.

Recommended follow-up: investigate whether the bot-autopilot
refactor (PR #79) or the loot-recovery feature (PR #85) bumped
baseline RSS along the soak code paths.  These don't affect
gameplay -- they're observability regressions.
