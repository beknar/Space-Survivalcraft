## Triage

### Unit suite — 1 pre-existing RSS flake

`test_soak_base_gc.py::TestGetRssMb::test_repeated_calls_return_close_values`
fails in mixed unit runs and **passes in isolation**.  Documented
in every TEST_RESULTS since PR #75 — `abs(rss_a - rss_b) < 5 MB`
assertion vs natural RSS variance under load (here 83 MB swing).
Not a regression.

### Soak suite — 1 marginal threshold failure

`TestSoakBasicShipRebuild::test_basic_ship_rebuild_5min_soak`
failed with **78.4 MB memory growth vs 75 MB threshold** — a
3.4 MB overage.  Same environmental-variance flake pattern
documented in the 2026-05-10 cycle (PR #91 had to raise these
thresholds for exactly this kind of noise).  Other 70 soak
tests pass.  Not a regression.

## Soak suite (71 tests, 70 PASSED + 1 marginal threshold)

The `Basic Ship rebuild loop` soak test exceeded the 75 MB memory
threshold by 3.4 MB (78.4 actual).  This is the same
environmental-variance flake pattern that PR #91 documented:
pymalloc arena retention + Arcade SpriteList growth can swing
memory by a few MB across runs.  The 70 other soak tests pass
under the PR #91 thresholds.

## Notes

* The integration + soak run was a single `pytest "unit tests/integration/" -s`
  invocation.  Logs: `_run_unit_2026-05-13_22-35.txt`,
  `_run_integration_fps_2026-05-13_22-35.txt`.
* This cycle delivered the largest perf-mean improvement
  observed across the multi-PR boss-fight tuning series
  (231.7 mean FPS vs 216.1 in the prior cycle, +7 %).  Likely
  driven by the engage_boss decomposition (PR #111) which
  moved per-tick boss-fight work into smaller, more inline-able
  helpers.
