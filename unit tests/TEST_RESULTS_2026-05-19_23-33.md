### VideoPlayer play/stop churn — primary concern

13.8 FPS is the only sample BELOW the 60 FPS floor this cycle.
The context exercises rapid play/stop transitions on the FFmpeg
backend.  Last cycle it ran at 239.8 FPS, so a ~94 % regression
in one place strongly suggests a specific change touching the
video subsystem.  Nothing in the bot-autopilot PRs (#150-#155)
should have hit the video path — worth bisecting against the
underlying gap (PRs #149-#155 + any commits not behind a PR).

invulnerable as new regressions.  All three recovered (or
substantially recovered) this cycle.  Consistent with run-to-run
variance on the boss-tick code path.
