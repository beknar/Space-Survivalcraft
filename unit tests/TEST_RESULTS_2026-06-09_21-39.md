### Basic Ship rebuild soak — memory marginal (host artifact, bisected)

`test_basic_ship_rebuild_5min_soak` failed in-suite with **90.9 MB
growth vs the 75 MB threshold** (FPS healthy throughout: 234–306).

| Run | Commit | Machine state | Result |
|---|---|---|---|
| In-suite | `77d1e41` (main) | hour ~5.4 of the suite | **FAIL** 90.9 MB |

| Probe | `8af1a85` (pre-planets) | settled | PASS |
| Probe | `9f21f2b` (planets merge) | settled | PASS |
| Isolated #2 | `77d1e41` (main) | settled | **PASS** (394 MB end, best of all runs) |

The failures only reproduce under host load; three settled runs pass
comfortably across main and both ancestor commits.  Code-wise, the
only commits unique to main vs the passing `9f21f2b` probe are the
four death-chain PRs (#238–#242) — all bot-process-only modules the
game never imports during a soak, so a code-driven RSS delta is not
plausible.  **Verdict: host-load-dependent marginal, not a
regression.**  Carry forward: if it recurs in the next cycle, bump the
rebuild soak's threshold from 75 to ~100 MB to match observed
host variance.
